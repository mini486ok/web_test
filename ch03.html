인터프리터를 띄워 봅시다

새 PowerShell 창을 열고 다음 명령을 입력합니다.

이 명령을 실행하면 다음과 비슷한 메시지가 보입니다:

>>> 가 보이면 성공이에요. 이게 Python의 대화형 인터프리터(interactive interpreter) 입니다. 한 줄씩 코드를 입력하면 즉시 답을 돌려줘요. 마치 계산기처럼 쓸 수 있습니다.

"인터프리터"라는 단어가 어렵다면

통역사라는 뜻이에요. 우리가 쓴 Python 코드를 컴퓨터 언어로 통역해서 즉시 실행해주는 사람(프로그램)이라고 보면 됩니다.

변수와 자료형 다섯 가지

>>> 다음에 한 줄씩 입력해 봅시다. 엔터를 누르면 즉시 결과가 보입니다.

엔터를 누르면 아무것도 안 보이는 게 정상이에요. 변수에 숫자를 담기만 했으니까요. 이번엔 a를 그냥 쳐 봅시다.

방금 우리는 변수 a에 정수 3을 담고, 다시 꺼내본 거예요. 같은 식으로 다섯 가지 자료형을 한 번에 만나봅시다.

각 줄은 차례대로:

  • 정수(int) — 42 같은 소수점 없는 숫자
  • 실수(float) — 3.14 같은 소수점 있는 숫자
  • 문자열(str) — 따옴표로 감싼 글자
  • 리스트(list) — 대괄호 [ ] 안에 여러 값을 순서대로 담은 것
  • 딕셔너리(dict) — 중괄호 { } 안에 "키: 값" 쌍을 담은 것

각 변수를 그대로 호출해 보세요.

따옴표 종류에 주의

코드 안의 따옴표는 반드시 곧은 따옴표(" 또는 ')여야 해요. 한글 워드프로세서가 자동으로 만드는 곡선 따옴표(", ')를 복사해 넣으면 SyntaxError가 납니다. 가능하면 직접 키보드에서 입력하세요.

함수 정의 — 가장 중요한 한 단락

이제 가이드 전체에서 가장 중요한 문법인 함수 정의 입니다. Chapter 4에서 우리가 만들 MCP 도구가 바로 이 형태예요.

>>> 다음에 한 줄씩 입력해 봅시다.

또 한 번 4칸 띄우고:

이번엔 4칸 들여쓰기 없이 그냥 엔터만 누르세요. 함수 정의가 끝났다는 신호입니다. >>> 가 다시 나타납니다.

이제 함수를 호출해봅시다.

성공입니다. 우리가 방금 만든 건 두 정수를 받아 합을 돌려주는 함수예요.

들여쓰기는 반드시 같은 폭으로

Python은 들여쓰기로 함수 본문을 구분합니다. 한 줄은 4칸, 다른 줄은 2칸으로 섞이면 IndentationError가 나요. 가장 안전한 방법은 항상 스페이스 4칸입니다. 탭과 스페이스를 섞지 마세요.

방금 입력한 코드 한 줄씩 해석

  • def add(a: int, b: int) -> int:

    • def는 "지금부터 함수를 정의해"라는 키워드.
    • add는 함수 이름.
    • 괄호 안의 a: int, b: int는 인자(파라미터). 콜론 뒤의 int는 타입 힌트예요. "이 인자는 정수여야 한다"는 표시.
    • -> int는 반환 타입 힌트. "이 함수는 정수를 돌려준다"는 표시.
    • 끝에 콜론(:)이 와야 합니다.

  • """Add two numbers together."""

    • 큰따옴표 세 개로 감싼 문자열을 함수의 첫 줄에 두면 docstring(문서 문자열)이 돼요.
    • 사람도 읽고, 나중에 LLM도 읽습니다. Chapter 4에서 매우 중요한 역할을 합니다.

  • return a + b

    • return 뒤의 값을 함수의 결과로 돌려줘요.

타입 힌트가 왜 중요한가

Python은 타입 힌트가 없어도 동작해요. 그런데 우리가 만들 MCP 도구에서는 타입 힌트가 LLM에게 도구의 사용 설명서를 자동으로 만들어주는 핵심 재료가 됩니다. MCP SDK가 타입 힌트를 보고 "이 함수는 정수 두 개를 받아서 정수 하나를 반환한다"는 스키마를 자동으로 만들어요. 그러니까 우리는 일부러라도 모든 인자에 타입 힌트를 답니다.

import — 다른 사람의 코드 가져오기

Python에는 이미 만들어진 코드 묶음이 잔뜩 있어요. 그걸 우리 코드에서 쓰려면 import라는 단어를 사용합니다.

엔터를 누르면 아무 메시지도 안 나와요. 정상입니다. 이제 math 안의 함수를 쓸 수 있어요.

sqrt는 제곱근(square root)을 계산하는 함수예요. math.sqrt(16)은 "math 모듈 안의 sqrt 함수에 16을 넣어 호출"하는 뜻입니다.

모듈, 패키지, 라이브러리

비슷한 단어가 많지만 결국 다 "다른 사람이 만든 코드 묶음"이에요. import math는 Python에 기본 포함된 표준 모듈이고, 우리가 Chapter 4에서 만날 mcp는 따로 설치한 외부 패키지입니다. 둘 다 import로 가져옵니다.

인터프리터 종료하기

이제 인터프리터를 정상적으로 종료해 봅시다. 다음 둘 중 아무거나 입력하면 됩니다.

또는 키보드에서 Ctrl + Z를 누른 뒤 엔터.

다시 PowerShell의 일반 프롬프트(PS C:\...>)가 보이면 정상 종료된 거예요.

인터프리터에서 빠져나오는 다른 방법을 모르겠을 때

  • Ctrl + C를 두 번 누르면 강제 인터럽트.
  • 그래도 안 되면 PowerShell 창 자체를 닫고 새로 열면 됩니다.

가상환경(virtual environment)은 뭐고 왜 필요한가

이 단어는 들어두기만 하세요. 직접 만들 일이 거의 없습니다. 우리는 uv가 자동으로 처리하게 둘 거예요.

가상환경은 한 줄로 말하면 "프로젝트마다 따로 쓰는 자기만의 작은 Python 상자" 입니다. A 프로젝트는 mcp 1.5 버전을 쓰고, B 프로젝트는 mcp 2.0 버전을 쓰고 싶을 때, 두 프로젝트가 서로 충돌하지 않도록 각자의 상자에 패키지를 따로 담는 거예요.

옛날에는 가상환경을 직접 만들고 활성화해야 했지만(venv, pip...), uv는 이 모든 걸 알아서 합니다. 우리가 Chapter 4에서 uv add "mcp[cli]" 한 줄을 치면 uv가 알아서 가상환경을 만들고, 그 안에 mcp를 설치해줘요. 우리가 신경 쓸 건 없습니다.

"가상환경"이라는 단어를 보고 어렵다고 느낄 필요 없어요. 이 가이드 안에서는 단 한 번도 직접 만들지 않습니다. uv가 다 합니다.
