"Chapter 4에서 한 게 사실 이거"

Chapter 4의 마지막 줄을 다시 봅시다.

transport="stdio". 이게 stdio 전송이에요. 좀 더 정확히 그림을 그리면 다음과 같습니다.

요약하면:

  • Claude Desktop이 우리 server.py를 자식 프로세스로 직접 띄웁니다.
  • Claude Desktop이 메시지를 보낼 때는 우리 프로세스의 표준 입력(stdin) 에 JSON 한 줄을 씁니다.
  • 우리 서버의 응답은 표준 출력(stdout) 으로 한 줄씩 흘러나옵니다.
  • 로그나 디버그 메시지는 표준 에러(stderr) 로 보내야 해요. stdout에 다른 글자를 쓰면 메시지 프레임이 깨집니다.

stdin / stdout / stderr가 뭔가요?

모든 프로세스는 기본적으로 세 개의 통로를 가져요. stdin(표준 입력)은 외부에서 들어오는 글자, stdout(표준 출력)은 프로그램이 내보내는 결과 글자, stderr(표준 에러)는 별도로 분리된 에러/로그 글자예요. 보통 PowerShell은 stdout과 stderr를 같은 화면에 같이 보여주지만, 기술적으로는 다른 통로입니다.

언제 Stdio를 쓰는가

한 줄: 같은 컴퓨터의 클라이언트(Claude Desktop, Cursor, Claude Code 등)에 도구를 붙일 때.

Stdio의 특징:

  • 인증 불필요 — 같은 사용자의 프로세스이기 때문에 외부에 노출이 없음
  • 즉시 동작 — 추가 설정 없이 자식 프로세스만 띄우면 끝
  • 한 클라이언트 전용 — 한 번에 한 클라이언트만 이 서버를 점유
  • 원격 공유 불가 — 인터넷 너머의 친구가 내 서버를 호출할 수 없음
  • 클라이언트가 죽으면 서버도 죽음 — 부모 프로세스가 종료되면 자식도 종료

측면장점단점
