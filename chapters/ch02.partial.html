이 챕터는 가이드 전체에서 가장 사고가 많이 나는 구간이에요. 그래서 25분으로 넉넉하게 잡았습니다. 막히면 빨간색 "막힘 회복" 박스를 먼저 보세요. 명령어는 모두 그대로 복사해서 쓸 수 있게 적어 두었습니다.

이 챕터에서 설치할 것

  • Python 3.11 이상 — MCP 서버 코드를 돌리는 언어
  • Windows Terminal — 명령어를 입력하는 검은 창
  • uv — Python 패키지·가상환경을 자동으로 관리해주는 마법 같은 도구
  • (선택) VS Code — 코드를 편하게 편집하는 에디터

이 챕터에서는 "터미널", "PATH", "패키지 매니저"라는 단어가 처음 등장합니다

  • 터미널: 컴퓨터에게 글자로 명령을 내리는 검은 창. Windows에서는 PowerShell이 그 안에 떠 있습니다.
  • PATH: "이 폴더에 있는 프로그램은 그냥 이름만 쳐도 실행해도 돼"라고 운영체제에 알려둔 폴더 목록.
  • 패키지 매니저: Python 라이브러리(다른 사람이 만든 코드 묶음)를 자동으로 받아오고 정리해주는 도구. uv가 이 역할을 합니다.

1작업 폴더 만들기

먼저 우리 가이드를 진행할 폴더를 하나 정합니다. 꼭 영문 경로에 만드세요. 한글이나 공백이 들어간 경로는 나중에 이상한 에러를 만듭니다.

키보드에서 Win + E를 눌러 파일 탐색기를 엽니다. 주소창에 C:\를 입력하고 엔터를 누르세요. C 드라이브 최상단으로 이동했죠? 빈 곳에서 우클릭 → "새로 만들기" → "폴더"를 선택하고 이름을 mcp-lab이라고 적습니다.

이게 앞으로 우리 작업 폴더예요. 이 챕터에서는 이 폴더 안에 아무것도 안 만들지만, Chapter 4부터 본격적으로 사용합니다.

한글 사용자명 함정

Windows 사용자명이 한글(예: 홍길동)이면 C:\Users\홍길동\Desktop\mcp-lab처럼 한글이 섞인 경로가 생길 수 있어요. 이런 경로는 가상환경이 깨지는 원인이 됩니다. 그래서 일부러 C:\mcp-lab처럼 사용자 폴더 바깥에 만드는 거예요.

2Python 설치

이제 Python을 설치합니다. 웹 브라우저(Edge, Chrome 등)를 열고 아래 주소로 갑니다.

페이지 상단의 노란색 큰 버튼 "Download Python 3.14.x"(2026년 4월 기준 최신 안정판)를 누르세요. python-3.14.x-amd64.exe 같은 파일이 다운로드 됩니다.

다운로드한 파일을 더블클릭하면 설치 마법사가 뜹니다. 여기서 가장 중요한 한 가지가 있습니다.

절대 빠뜨리면 안 되는 체크박스

설치 마법사 첫 화면 하단에 작은 글씨로 "Add python.exe to PATH" 라는 체크박스가 있습니다. 반드시 이걸 먼저 체크하세요. 이걸 빼먹으면 새 터미널에서 python 명령이 인식되지 않아요. 초보자가 가장 자주 겪는 1번 함정입니다.

체크를 한 뒤 "Install Now" 를 누릅니다. 설치는 1~2분 걸려요. "Setup was successful"이라는 메시지가 나오면 "Close" 로 닫으세요.

더 깔끔한 옵션: Python Install Manager

공식적으로는 2026년부터 Python Install Manager(pymanager)라는 새 도구를 권장하기도 합니다. 여러 Python 버전을 관리해야 하는 고급 사용자라면 winget install 9NQ7512CXL7T 한 줄로 설치할 수 있어요. 초보자는 그냥 위의 전통 인스톨러로도 충분합니다.

PowerShell vs CMD

Windows Terminal 안에서는 여러 종류의 셸을 띄울 수 있어요. 우리는 처음부터 끝까지 PowerShell(파란 배경, PS로 시작)을 사용합니다. CMD(C:\> 로 시작하는 검은 창)는 명령어가 살짝 다르므로 헷갈리지 않도록 PowerShell만 쓰세요.

  • "Add python.exe to PATH" 체크박스를 빼먹은 경우입니다. 가장 흔해요.
  • 해결: 시작 메뉴 → "프로그램 추가/제거" → Python 항목 → "수정(Modify)" → "Next" → 두 번째 화면에서 "Add Python to environment variables" 체크 → "Install" 클릭.

where.exe python의 결과가 Microsoft Store 경로면

C:\Users\사용자이름\AppData\Local\Microsoft\WindowsApps\python.exe 같은 게 보이면 Microsoft Store의 가짜 Python이 잡힌 거예요. 이건 진짜 Python이 아닙니다.

  • 시작 메뉴 → "앱 별칭 관리(Manage app execution aliases)" 검색.
  • 목록에서 "App Installer (python.exe)"와 "App Installer (python3.exe)"를 모두 꺼짐(Off) 으로 바꾸세요.
  • PowerShell을 닫았다 다시 열고 where.exe python을 다시 실행. 이제 C:\Users\...\Programs\Python\... 경로가 보일 거예요.

5uv 설치

이제 가이드 전체를 통틀어 가장 중요한 도구, uv를 설치합니다.

uv는 한마디로 "가상환경과 패키지를 자동으로 관리해주는 마법 상자" 예요. 가상환경(virtual environment)이 뭔지 지금 몰라도 괜찮아요. uv가 알아서 만들고, 알아서 활성화하고, 알아서 패키지를 넣어줍니다. 우리는 그냥 uv add, uv run 두 명령만 외우면 됩니다.
