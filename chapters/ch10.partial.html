동료에게 공유하기 — git clone + uv sync 패턴

가장 간단하면서 가장 자주 쓰이는 공유 방법은 GitHub에 올리고 동료가 git clone하는 거예요. uv를 함께 쓰면 동료는 단 두 줄로 우리 서버를 자기 PC에서 그대로 띄울 수 있어요.

준비 1: README.md

C:\mcp-lab\hello-mcp\README.md를 다음 최소 항목으로 채웁니다. (아래는 README.md 파일 안에 들어갈 내용 예시입니다. 그대로 복사해 README.md에 붙여 넣으세요.)

이게 최소한이에요. 동료가 따라할 수 있도록 4가지가 들어가야 합니다:

  1. 무엇인지 한 줄
  2. 사전 요구사항
  3. 설치 명령
  4. 실행 / 등록 방법

(선택) PyPI 업로드는 어떤 거예요?

PyPI는 Python 패키지 공식 저장소예요. pip install hello-mcp 한 줄로 누구나 우리 서버를 설치할 수 있게 만드는 옵션입니다.

작은 사이드 프로젝트라면 굳이 PyPI에 올릴 필요는 없어요. 동료 몇 명과 공유한다면 git clone 방식이 더 쉽고 빠릅니다. PyPI는 다음 같은 상황에서 고려하세요:

  • 모르는 사람에게 공개적으로 제공할 때
  • uvx hello-mcp 한 줄로 설치 없이 실행 가능하게 만들고 싶을 때
  • 다른 패키지의 의존성으로 들어가야 할 때

PyPI 업로드는 uv build → uv publish 두 명령으로 가능해요. 자세한 절차는 uv 공식 문서를 참고하세요.

원격 호스팅 옵션 (이름만)

Streamable HTTP 서버를 인터넷 너머에 띄우고 싶다면 다음 같은 옵션이 있어요. 이 가이드는 깊이 다루지 않지만, 검색할 키워드만 적어 둡니다.

  • Cloudflare Workers / Cloudflare Tunnel — 무료 티어, 서버리스
  • fly.io — 가벼운 컨테이너 호스팅
  • Render / Railway — 간단한 PaaS
  • 자체 서버 — VPS에 직접 띄우기

원격에 띄우는 순간 보안이 시작됩니다

Chapter 8에서 강조한 보안 3원칙(127.0.0.1 바인딩 / Origin 검증 / 인증)을 다시 보고 시작하세요. 인증 없이 인터넷에 띄우면 누구나 호출할 수 있어요. 모델이 비용 청구식이라면 청구액이 금방 폭발합니다.

트러블슈팅 표 — 가장 많이 만나는 7가지

#증상원인 후보확인 명령 / 위치해결
1python --version이 안 됨"Add to PATH" 미체크, 또는 새 터미널 안 엶where.exe python인스톨러 다시 실행 → "Modify" → "Add Python to environment variables" 체크
2uv --version이 안 됨설치 후 같은 창에서 실행ls $env:USERPROFILE\.local\bin새 PowerShell 창을 열기
3Claude Desktop에서 도구가 안 보임설정 파일 JSON 문법 오류%APPDATA%\Claude\claude_desktop_config.json을 VS Code로 열기JSON 빨간 줄 수정. 백슬래시 두 개. 콤마 확인
4"uv command not found" 로그Claude Desktop과 PATH 다름%APPDATA%\Claude\logs\mcp-server-*.logcommand를 uv 절대 경로로 변경 (where.exe uv 결과)
5도구는 보이는데 호출 시 에러server.py에 print 사용로그 파일에서 JSON 파싱 에러 확인모든 print 제거 또는 file=sys.stderr 추가
6타입 에러 (Pydantic validation)인자 타입 힌트와 LLM이 보낸 값 불일치로그 파일의 validation errordocstring과 description을 더 명확히
7Streamable HTTP에서 포트 충돌8000 포트 사용 중netstat -ano | findstr :8000FastMCP("name", port=8765)로 포트 변경

표에 내 증상이 없을 때

Claude Desktop 로그가 가장 먼저 봐야 할 곳이에요.

  1. 파일 탐색기 주소창에 %APPDATA%\Claude\logs 입력
  2. mcp-server-<서버이름>.log 파일을 메모장이나 VS Code로 열기
  3. 마지막 50줄을 읽기. 에러 메시지의 키워드(예: "ModuleNotFoundError", "Connection refused")를 GitHub의 modelcontextprotocol/python-sdk 리포지토리 Issues 검색에 그대로 붙여넣어 보세요.

Claude Desktop이 아닌 다른 클라이언트도 같은 방식

이 가이드는 Claude Desktop을 기준으로 진행했지만, MCP는 표준이라서 다른 클라이언트도 거의 똑같은 방식으로 등록됩니다.

  • Cursor: Settings → MCP → Add MCP server. JSON 구조는 Claude Desktop과 거의 동일.
  • Claude Code (VS Code 확장): 설정에서 MCP 서버 등록.
  • VS Code 빌트인 MCP 지원: 최근 버전에서 추가됨.

한 번 만든 서버를 여러 클라이언트에 동시에 등록할 수 있어요. 어떤 클라이언트에 있든 같은 도구가 같은 결과를 돌려줍니다. 이게 MCP가 표준인 이유의 본질이에요.
