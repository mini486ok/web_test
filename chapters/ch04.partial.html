왜 따옴표를 쓰나요?

mcp[cli]의 대괄호([ ])는 PowerShell이 특수 문자로 해석할 수 있어요. 그래서 통째로 큰따옴표로 감싸 "mcp[cli]"라고 적습니다. 대괄호 안의 cli는 "기본 mcp 외에 cli 도구도 같이 깔아줘"라는 뜻이에요.

설치가 끝나면 폴더 안에 .venv 디렉토리와 uv.lock 파일이 새로 생겼을 거예요. .venv가 바로 Chapter 3에서 말한 가상환경입니다. uv가 알아서 만들어줬어요. 우리는 한 번도 직접 활성화하지 않습니다.

한 줄씩 해석

  • from mcp.server.fastmcp import FastMCP — MCP SDK에서 FastMCP라는 도구 상자를 가져옵니다.
  • mcp = FastMCP("Hello MCP") — "Hello MCP"라는 이름의 서버 한 개를 만듭니다.
  • @mcp.tool() — 바로 아래의 함수를 MCP 도구로 등록한다는 표시. 괄호를 꼭 빼먹지 마세요. @mcp.tool(괄호 없이)로 적으면 안 됩니다.
  • def add(a: int, b: int) -> int: — Chapter 3에서 본 그 함수예요. 타입 힌트가 있어야 MCP가 LLM에 보여줄 스키마를 자동으로 만들 수 있습니다.
  • """Add two numbers together. ...""" — docstring. 이 글을 LLM이 직접 읽어서 "이 도구를 언제 써야 하는지" 판단합니다. 대충 쓰면 LLM이 도구를 잘못 사용합니다.
  • if __name__ == "__main__": — "이 파일이 직접 실행됐을 때만 아래를 실행해줘"라는 Python 관용구.
  • mcp.run(transport="stdio") — 서버를 stdio 방식으로 시작. stdio가 뭔지는 Chapter 7에서 자세히 다룹니다. 지금은 "Claude Desktop이 우리 서버를 직접 띄울 때 쓰는 방식"이라고만 알아두세요.

print를 추가하지 마세요

디버깅 목적으로 print("hello")를 함수 안에 넣고 싶을 수 있어요. stdio 모드에서는 print 금지입니다. Claude Desktop과 우리 서버는 stdout(표준 출력)으로 JSON 메시지를 주고받는데, 거기에 우리가 추가한 print 글자가 끼면 메시지가 깨져서 모든 도구가 멈춰요. 로그가 필요하면 Chapter 7에서 알려주는 stderr 방식을 쓰세요.

4단독으로 한 번 띄워보기

진짜로 Claude에 붙이기 전에, 서버가 import 에러 없이 잘 시작하는지 확인합시다.

이 명령을 실행하면 화면에 아무것도 보이지 않습니다. 그게 정상이에요. stdio 모드의 서버는 stdin에서 메시지를 기다리며 가만히 있습니다. 에러 메시지가 안 나오면 import 단계는 통과한 거예요.

확인이 끝났으면 Ctrl + C를 눌러 종료합니다.

uv run에서 에러가 보이면

  • ModuleNotFoundError: No module named 'mcp' 가 보이면 → uv add "mcp[cli]"를 다시 실행하세요. 현재 폴더가 C:\mcp-lab\hello-mcp인지 확인.
  • IndentationError 또는 SyntaxError 가 보이면 → server.py의 들여쓰기를 확인. 모든 함수 본문은 스페이스 4칸으로 시작해야 합니다.
  • SyntaxError: invalid character 가 보이면 → 코드 안의 따옴표가 곡선 따옴표일 가능성. 위의 코드 블록에서 "복사" 버튼으로 다시 가져오세요.

핵심 한 가지: 백슬래시는 두 개

Windows 경로에는 백슬래시(\)가 들어가요. 그런데 JSON 파일 안에서는 백슬래시 한 개가 특별한 의미를 가지기 때문에, 반드시 두 개로 적어야 합니다.

  • 잘못된 예: "C:\mcp-lab\hello-mcp" → 파싱 에러
  • 올바른 예: "C:\\mcp-lab\\hello-mcp"

8Claude Desktop 완전 종료 후 재시작

설정 파일을 저장했다고 바로 반영되지 않아요. Claude Desktop을 완전히 종료한 다음 다시 켜야 합니다.

"완전히"가 중요해요. 단순히 X 버튼으로 창을 닫으면 트레이(작업 표시줄 우하단의 작은 아이콘 모음)에 숨어 있을 뿐입니다. 그 상태로 다시 켜면 변경사항이 안 읽혀요.

  1. 작업 표시줄 우하단의 위쪽 화살표(^)를 클릭해 트레이를 펼칩니다.
  2. Claude 아이콘을 우클릭 → Quit 또는 종료 를 선택합니다.
  3. 트레이에서 Claude 아이콘이 사라졌는지 확인합니다.
  4. 시작 메뉴에서 Claude Desktop을 다시 켭니다.

트레이를 꼭 확인하세요

"닫았다고 생각했는데 안 닫혔던 케이스"가 정말 자주 일어납니다. 트레이 아이콘이 사라진 걸 두 번 확인하세요.

10잘 안 될 때 — 가장 흔한 3가지

만약 슬라이더 아이콘이 안 보이거나, 보여도 호출이 실패한다면 아래 세 가지를 차례로 점검하세요.

1. JSON 문법 오류

claude_desktop_config.json을 다시 열어보세요. 가장 흔한 실수는:

  • 마지막 객체에 콤마(,)가 잘못 들어감
  • 백슬래시가 한 개만 들어감
  • 큰따옴표가 곡선 따옴표(")로 바뀌어 있음

JSON 검증을 위해 VS Code로 파일을 열면 문제가 있는 줄에 빨간 줄이 그어져요. 또는 브라우저에서 https://jsonlint.com 에 내용을 붙여넣어 검증할 수 있습니다.

2. uv를 찾지 못함

Claude Desktop의 로그를 확인합니다. 파일 탐색기 주소창에 다음을 입력하세요:

폴더 안에 mcp-server-hello-mcp.log 파일이 있을 거예요. 메모장으로 열어 마지막 몇 줄을 보세요. "uv: command not found" 또는 "The system cannot find the file specified" 같은 줄이 보이면 위의 막힘 회복 박스(uv가 PATH에 없다는 에러)에 따라 절대 경로 방식으로 바꾸세요.

3. 서버가 stdout을 오염시킴

만약 우리 코드에 print(...)를 추가했다면 모든 도구 호출이 실패합니다. server.py에서 print 호출을 모두 지우고 Claude Desktop을 완전히 재시작하세요.

4. 그래도 안 된다면 PowerShell에서 서버를 직접 띄워보기

Claude Desktop이 시도하는 명령과 같은 것을 PowerShell에서 직접 실행해보면 어디서 막히는지 보입니다.

에러 메시지가 보이면 그게 진짜 원인이에요. 아무 메시지 없이 멈춰 있으면 정상 (stdio 서버라 stdin을 기다리는 중). Ctrl + C로 종료.
